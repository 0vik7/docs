# Доступ

По умолчанию все операции в [!KEYREF ig-name] выполняются от имени сервисного аккаунта.

Сервисный аккаунт — это специальная учетная запись, которая может использоваться службами и приложениями для взаимодействия с другими API Яндекс.Облака от вашего имени. Подробнее про сервисные аккаунты и управление доступом в Яндекс.Облаке читайте в документации сервиса [[!KEYREF iam-full-name]](../../../iam/).

Приложения [!KEYREF ig-name] используют [авторизационные ключи](../../../iam/concepts/authorization/key.md) сервисного аккаунта для авторизации в наборе API и выполняют операции с ресурсами в пределах ролей, предоставленных сервисному аккаунту. Например, [!KEYREF ig-name] использует сервисный аккаунт для создания, обновления и удаления виртуальных машин. [!KEYREF ig-name] легко аутентифицируется в API [!KEYREF compute-name] без встраивания секретных ключей или учетных данных пользователя в код приложения.

Группа виртуальных машин может иметь только один сервисный аккаунт, созданный в том же каталоге, что и группа.
