# Метод get
Возвращает указанный ресурс Instance.
 
Чтобы получить список доступных ресурсов Instance, используйте
запрос [list](/docs/compute/api-ref/Instance/list).
 
## HTTP-запрос
`GET /compute/v1/instances/{instanceId}`
 
## Path-параметры {#path_params}
 
Name | Description
--- | ---
instanceId | Обязательное поле. Идентификатор возвращаемого ресурса Instance. Чтобы получить идентификатор виртуальной машины, используйте запрос [list](/docs/compute/api-ref/Instance/list).  Максимальная длина — 50 символов.
 
## Query-параметры {#query_params}
 
Name | Description
--- | ---
view | Определяет, какая информация о ресурсе Instance должна быть возвращена в ответе сервера.
 
## Ответ {#responses}
**HTTP Code: 200 - OK**

Ресурс Instance. Дополнительные сведения см. в разделе
[Виртуальные машины](/docs/compute/concepts/vm).
 
Поле | Описание

secondaryDisks.<br>mode | **string**<br>Режим доступа к ресурсу Disk.   - READ_ONLY: Доступ на чтение.  - READ_WRITE: Доступ на чтение и запись.<br>`READ_ONLY`<br>`READ_WRITE`<br>

networkInterfaces.<br>macAddress | **string**<br>Только для вывода. MAC-адрес, назначенный сетевому интерфейсу.
