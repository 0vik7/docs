# Метод get
Возвращает указанный ресурс Snapshot.
 
Чтобы получить список доступных ресурсов Snapshot, используйте
запрос [list](/docs/compute/api-ref/Snapshot/list).
 
## HTTP-запрос
`GET /compute/v1/snapshots/{snapshotId}`
 
## Path-параметры {#path_params}
 
Name | Description
--- | ---
snapshotId | Обязательное поле. Идентификатор возвращаемого ресурса Snapshot. Чтобы получить идентификатор снимка, используйте запрос [list](/docs/compute/api-ref/Snapshot/list).  Максимальная длина — 50 символов.
 
## Ответ {#responses}
**HTTP Code: 200 - OK**

Ресурс Snapshot. Дополнительные сведения см. в разделе
[Снимки дисков](/docs/compute/concepts/snapshot).
 
Поле | Описание
--- | ---
id | **string**<br><p>Только для вывода. Идентификатор снимка.</p> 
folderId | **string**<br><p>Идентификатор каталога, которому принадлежит снимок.</p> 
createdAt | **string** (date-time)<br><p>Только для вывода. Время создания ресурса в формате в <a href="https://www.ietf.org/rfc/rfc3339.txt">RFC3339</a>.</p> 
name | **string**<br><p>Имя снимка. Длина 1-63 символов.</p> 
description | **string**<br><p>Описание снимка. Длина 0-256 символов.</p> 
labels | **object**<br><p>Метки ресурса в формате ключ-значение. Максимум 64 на ресурс.</p> 
storageSize | **string** (int64)<br><p>Только для вывода. Размер снимка в байтах.</p> 
diskSize | **string** (int64)<br><p>Только для вывода. Размер диска в байтах в момент создания снимка.</p> 
productIds | **string**<br><p>Только для вывода. Идентификаторы лицензий, указывающие, какие лицензии подключены к этому ресурсу. Идентификаторы лицензий используются для расчета дополнительной платы за использование виртуальной машины.</p> <p>Правильный идентификатор лицензии генерируется Яндекс.Облаком. Новые ресурсы, созданные из этого ресурса, унаследуют идентификаторы от него.</p> <p>Если вам известны идентификаторы лицензий, укажите их при создании образа. Например, если вы создадите образ диска с помощью сторонней утилиты и загрузите его в Yandex Object Storage, идентификаторы лицензий будут потеряны. Вы можете указать их в запросе<a href="/docs/compute/api-ref/Image/create">create</a>.</p> 
status | **string**<br><p>Только для вывода. Текущий статус снимка.</p> <ul> <li>CREATING: Снимок создается.</li> <li>READY: Снимок готов к использованию.</li> <li>ERROR: Со снимком произошла ошибка, блокирующая работу.</li> <li>DELETING: Снимок удаляется.</li> </ul> 
sourceDiskId | **string**<br><p>Идентификатор диска из которого создан этот снимок.</p>
