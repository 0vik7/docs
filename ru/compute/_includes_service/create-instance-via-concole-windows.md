Чтобы создать виртуальную машину:

1. Откройте каталог, в котором будет создана виртуальная машина.
2. Нажмите на кнопку **Добавить ресурс**.
3. Выберите **Виртуальная машина**.
4. В поле **Имя** введите имя виртуальной машины.

    [!INCLUDE [name-format](../../_includes/name-format.md)]

5. Выберите [зону доступности](../../overview/concepts/geo-scope.md), в которой будет находиться виртуальная машина.
6. Выберите один из публичных [образов](../concepts/images.md) на базе операционной системы Windows.
7. В блоке **Тип машины** укажите необходимые [вычислительные ресурсы](../concepts/vm-types.md) виртуальной машины: количество и производительность vCPU и объем RAM.
8. В блоке **Сети** выберите, к какой подсети необходимо подключить виртуальную машину при создании.
9. При создании виртуальной машины в операционной системе будет автоматически создан пользователь `Administrator`. В поле **Пароль** задайте пароль для этого пользователя, с которым можно будет войти на виртуальную машину по RDP.
    Пароль должен соответствовать требованиям к сложности паролей Windows. Проверка сложности пароля выполняется в операционной системе при создании пользователя. Пока пароль не задан, доступ на виртуальную машину по RDP невозможен.
    > [!IMPORTANT]
    >
    > Не используйте легко подбираемые пароли. При создании виртуальной машине будет присвоен публичный IP адрес. Если ваш пароль смогут подобрать злоумышленники, они получат доступ с правами администратора к вашей виртуальной машине.
10. Нажмите кнопку **Создать ВМ**. Создание и инициализация виртуальной машины Windows могут занять некоторое время. После того, как она перейдет в статус `RUNNING`, запустится процесс инициализации операционной системы. Это может занять до 5 минут. Вы можете следить за статусом новой виртуальной машины в списке виртуальных машин вашего каталога.

При создании виртуальной машине назначаются IP-адрес и имя хоста (FQDN). Эти данные можно использовать для доступа к виртуальной машине по RDP.
