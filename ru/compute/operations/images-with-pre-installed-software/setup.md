# Первичная настройка программного обеспечения

Для виртуальных машин с LEMP, LAMP и Node.js первичную настройку выполнять не нужно. Предустановленное программное обеспечение готово к использованию.

Для виртуальных машин с Drupal, GitLab, Joomla и WordPress для первичной настройки используйте [веб-установщик](#web-setup).


## Настройка через веб-установщик {#web-setup}

Для запуска веб-установщика:
1. В блоке **Сеть** на странице виртуальной машины найдите публичный IP-адрес виртуальной машины.
1. Откройте адрес `http://<IP-адрес>` в браузере. Откроется окно авторизации.
    > [!NOTE]
    >
    > Если у вас не получилось подключиться к виртуальной машине по HTTP, повторите попытку через несколько минут. После того, как виртуальная машина перешла в статус `RUNNING`, запустился процесс инициализации программного обеспечения. Это может занять до 5 минут.
1. В окне авторизации:
    - В поле **Имя пользователя** введите:
        - `root` для виртуальной машины GitLab;
        - `admin` для других виртуальных машин.
    - В поле **Пароль** введите идентификатор виртуальной машины. Вы можете найти идентификатор в URL страницы виртуальной машины в консоли управления.
    После успешной авторизации веб-установщик запустится автоматически. Для настройки программного обеспечения следуйте рекомендациям производителя: [Drupal](https://drupal.org/), [GitLab](https://about.gitlab.com), [Joomla](https://joomla.org/), [WordPress](https://wordpress.org/).

В процессе настройки вам могут понадобиться параметры подключения к базе данных. Чтобы узнать их, [подключитесь к виртуальной машине](operate.md) по протоколу SSH и выполните команду:
```
sudo cat /root/default_passwords.txt
```
