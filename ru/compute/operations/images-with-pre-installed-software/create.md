# Создание виртуальной машины из образа

Вы можете создать виртуальную машину из образа так же, как любую другую виртуальную машину в [!KEYREF compute-short-name]:

1. Если у вас уже есть каталог в Яндекс.Облаке, откройте страницу этого каталога в консоли управления. Если каталога еще нет, создайте его:
    
    [!INCLUDE [create-folder](../../../_includes/create-folder.md)]
    
1. Нажмите значок ![](../../../_assets/plus-sign.svg) на плитке **Yandex Compute Cloud**.
1. В поле **Имя** введите имя виртуальной машины.
    Имя должно быть уникальным в рамках каталога. Имя может содержать строчные буквы латинского алфавита, цифры и дефисы. Первый символ должен быть буквой. Последний символ не должен быть дефисом. Максимальная длина имени — 63 символа.
1. Выберите [зону доступности](../../../overview/concepts/geo-scope.md), в которой будет находиться виртуальная машина.
1. Выберите образ виртуальной машины c программным обеспечением, которое вы хотите использовать.
1. Выберите объем ресурсов виртуальной машины — количество виртуальных ядер и объем виртуальной памяти.
    > [!NOTE]
    > Для создания виртуальной машины из образа **GitLab** необходимо не меньше 4 виртуальных ядер и 4 ГБ оперативной памяти.
1. Укажите данные для доступа на виртуальную машину.
1. Нажмите кнопку **Создать ВМ**.
    Создание виртуальной машины может занять некоторое время. Вы можете проверить статус новой виртуальной машины в списке виртуальных машин вашего каталога. Когда виртуальная машина перейдет в статус `RUNNING`, переходите к [первичной настройке программного обеспечения](setup.md).
