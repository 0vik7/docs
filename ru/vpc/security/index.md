# Безопасность и управление доступом

Пользователь Яндекс.Облака может выполнять только те операции над ресурсами, которые разрешены назначенными ему ролями. Пока у пользователя нет никаких ролей, почти все операции ему запрещены. 

Чтобы разрешить доступ к ресурсам сервиса Yandex Virtual Private Cloud (облачным сетям и подсетям), назначьте пользователю нужные роли из приведенного ниже списка. На данный момент роль может быть назначена только на родительский ресурс (каталог или облако), роли которого наследуются вложенными ресурсами. 

> [!NOTE]
>
> Подробнее о наследовании ролей читайте в разделе [Наследование прав доступа](../../resource-manager/concepts/resources-hierarchy.md#access-rights-inheritance) документации сервиса Yandex Resource Manager.
>

## Назначение ролей

Для управления облачными сетями пользователь должен иметь соответствующие полномочия в облаке и каталогах, в которых будут выполняться операции.

## Роли

Ниже перечислены все роли, которые учитываются при проверке прав доступа в сервисе Compute Cloud.

### Сервисные роли

Сервисные роли — роли, дающие доступ к ресурсам определенного сервиса. При проверке прав доступа к ресурсам VPC учитываются сервисные роли Resource Manager.

[!INCLUDE [cloud-roles](../../_includes/cloud-roles.md)]

### Примитивные роли

Примитивные роли можно назначать на любой ресурс в любом сервисе.

#### viewer
Пользователь с ролью `viewer` может смотреть списки облачных сетей и подсетей в каталоге.

#### editor
Пользователь с ролью `editor` может выполнять любые операции с облачными сетями и подсетями в каталоге: создавать, удалять и изменять их.

#### admin
Пользователь с ролью `admin` может выполнять любые операции с облачными сетями и подсетями в каталоге: создавать, удалять и изменять их, а также назначать роли другим пользователям.
