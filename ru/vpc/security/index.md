# Безопасность и управление доступом

В этом разделе перечислены необходимые роли, которыми должен обладать пользователь для выполнения операций с ресурсом. 

> [!NOTE]
>
> Права доступа к ресурсам наследуются от облака и каталога.
>

## Назначение ролей

Для управления облачными сетями пользователь должен иметь соответствующие полномочия в облаке и каталогах, в которых будут выполняться операции.

Чтобы дать пользователю полномочия: 

[!INCLUDE [grant-role-console](../../_includes/grant-role-console.md)]

## Разрешения в [!KEYREF vpc-short-name]

[!INCLUDE [cloud-roles](../../_includes/cloud-roles.md)]

#### viewer
Пользователь с ролью `viewer` может смотреть списки облачных сетей и подсетей в каталоге.

#### editor
Пользователь с ролью `editor` может выполнять любые операции с облачными сетями и подсетями в каталоге: создавать, удалять и изменять их.

#### admin
Пользователь с ролью `admin` может выполнять любые операции с облачными сетями и подсетями в каталоге: создавать, удалять и изменять их, а также назначать роли другим пользователям.
