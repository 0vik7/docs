# Взаимосвязь ресурсов сервиса

Основная сущность, которой оперирует сервис Yandex [!KEYREF full-service-name], — _кластер баз данных_. Кластер можно создать для любой поддерживаемой СУБД.

Каждый кластер состоит из одного или нескольких _хостов БД_ — виртуальных машин с развернутой СУБД. Хосты кластера могут находиться в разных зонах и даже разных регионах доступности. [Подробнее о географии Облака](../../overview/concepts/geo-scope.md).

- Кластер из нескольких хостов автоматически отказоустойчив — хосты могут заменять друг друга в качестве основной реплики кластера.
- Кластер из одного хоста обойдется дешевле, но очевидным образом менее надежен.

При создании кластера необходимо указывать _класс БД_ — шаблон виртуальной машины, согласно которому будут развертываться хосты кластера. Список доступных классов БД и их характеристики см. в разделе [[!TITLE]](instance-types.md).

Созданный в каталоге кластер БД доступен по сети для всех виртуальных машин, подключенных к этой же облачной сети. [Подробнее о работе сети](../../vpc/).
