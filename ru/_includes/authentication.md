# Аутентификация в API

Аутентификация — проверка подлинности пользователя. Ниже описаны шаги аутентификации в Яндекс.Облаке.

## Шаг 1. Авторизация на Яндексе {#auth-at-yandex}

Выполните вход на Яндекс с использованием вашего аккаунта. Подробнее см. в документе [Ваш аккаунт в Яндексе](https://yandex.ru/support/passport/auth.html).

## Шаг 2. Получение OAuth-токена {#get-oauth-token}

Чтобы начать взаимодействие с API Яндекс.Облака, необходимо получить OAuth-токен. Токен позволяет приложениям обращаться к сервисам Яндекса от имени пользователя.

Получить OAuth-токен можно в сервисе Яндекс.OAuth. Для этого перейдите по [ссылке](https://oauth.yandex.ru/authorize?response_type=token&client_id=1a6990aa636648e9b2ef855fa7bec2fb) и нажмите кнопку **Разрешить**.

Сохраните полученный OAuth-токен.

## Шаг 3. Обмен OAuth-токена на IAM-токен {#change-oauth-to-iam}

Для работы с API других сервисов в Яндекс.Облаке необходимо получить IAM-токен. Для этого используйте метод [create](../iam/api-ref/IamToken/create.md). В теле запроса передайте полученный OAuth-токен (в поле `yandexPassportOauthToken`). IAM-токен действителен в течение 12 часов. После истечения срока действия получите новый IAM-токен.

## Шаг 4. Использование IAM-токена {#request-with-iam}

Полученный IAM-токен позволяет выполнять запросы к сервисам Яндекс.Облака. IAM-токен необходимо передавать в заголовке `Authorization` для каждого запроса к API.

Полный формат записи заголовка следующий:

```
Authorization: Bearer [ваш IAM-токен]
```

Перед выполнением запроса Яндекс.Облако проверяет действительность токена и возможность выполнения запроса. Если пользователь не обладает необходимыми полномочиями, запрос не будет выполнен.
