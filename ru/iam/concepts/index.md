# [!KEYREF iam-full-name]

[!INCLUDE-NOTITLE [iam-def](../../_includes/iam-def.md)]

С помощью [!KEYREF iam-short-name] вы сможете:

* [предоставить доступ к ресурсам](#access);
* [управлять аккаунтами в Яндекс.Облаке](#accounts);
* [авторизоваться в Яндекс.Облаке](#auth).


## Доступ {#access}

Чтобы предоставить пользователю доступ к ресурсу, вы назначаете ему [роли](access-control/roles.md) на ресурс. Каждая роль состоит из набора разрешений, описывающих допустимые операции с ресурсом.

Перед тем, как выполнить операцию с ресурсом, например создать виртуальную машину, Яндекс.Облако отправляет в IAM запрос на проверку, разрешена ли эта операция. [!KEYREF iam-short-name] сравнивает список необходимых разрешений со списком разрешений пользователя, выполняющего операцию. Если какого-то из разрешений у пользователя нет, операция не будет выполнена и Яндекс.Облако вернет ошибку. Подробнее читайте в разделе [[!TITLE]](access-control/index.md).


## Аккаунты {#accounts}

Для идентификации пользователей, выполняющих операции с ресурсами, используются [аккаунты Яндекс.Паспорта](#passport) и [сервисные аккаунты](#sa).

> [!NOTE]
>
> Платежный аккаунт не используется для управления ресурсами в Яндекс.Облаке и не относится к сервису [!KEYREF iam-short-name]. Подробнее читайте в разделе [[!TITLE]](../../billing/concepts/billing-account.md) документации по биллингу.

### Аккаунт Яндекс.Паспорта {#passport}

_Аккаунт Яндекс.Паспорта_ — ваш аккаунт на Яндексе или [Яндекс.Коннекте](https://connect.yandex.ru). Аккаунт Яндекс.Паспорта необходим для управления ресурсами через [консоль управления](console.cloud.yandex.ru).

### Сервисный аккаунт {#sa}

[!INCLUDE [sa-def](../_includes_service/sa-def.md)]

Использование сервисных аккаунтов позволяет гибко настраивать права доступа к ресурсам для написанных вами программ. Подробнее читайте в разделе [[!TITLE]](users/service-accounts.md).


## Авторизация {#auth}

Чтобы [!KEYREF iam-short-name] смог проверить, обладает ли пользователь необходимыми правами, пользователь должен авторизоваться. Авторизация происходит по-разному в зависимости от типа аккаунта и используемого интерфейса. Подробнее об этом читайте в разделе [[!TITLE]](authorization/index.md).
