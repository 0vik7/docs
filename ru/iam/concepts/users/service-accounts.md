# Сервисные аккаунты

_Сервисный аккаунт_ — учетная запись, от имени которой написанная вами программа может выполнять действия в Яндекс.Облаке. Сервисный аккаунт может управлять другими ресурсами через API.

Сервисные аккаунты будут полезны для автоматизации управления большим количеством ресурсов. Например, для осуществления автоматического резервного копирования дисков.

В Yandex Object Storage сервисные аккаунты используются для доступа к ресурсам сервиса.

Сервисный аккаунт при этом является ресурсом: он создается внутри каталога и вы можете выдавать и отнимать права доступа на этот ресурс другим пользователям.


## Статические ключи доступа {#static-credentials}

Статические ключи доступа — это связка ключ-идентификатор используются в Yandex Object Storage для сервисные аккаунты используются для доступа к ресурсам сервиса.

Механизм статических ключей доступа используется в сервисе Yandex Object Storage. Эти ключи необходимы для доступа сервисного аккаунта к ресурсам Yandex Object Storage через API, совместимый с Amazon S3.

Статические ключи доступа представляют собой пару _id_ и _секретный ключ_.

Подробнее читайте в разделе [[!TITLE]](../../../storage/s3/index.md) документации сервиса Yandex Object Storage
