# Авторизация в Яндекс.Облаке

Когда пользователь выполняет операцию с ресурсом Яндекс.Облака, IAM проверяет, обладает ли пользователь необходимыми разрешениями для этой операции.

Разрешения пользователи получают вместе с ролями на ресурсы. Подробнее о том, как назначаются роли и проверяется список разрешений, читайте в разделе [[!TITLE]](../access-control/index.md).

Авторизация происходит по-разному в зависимости от типа аккаунта и используемого интерфейса.

## Авторизация с аккаунтом Яндекс.Паспорта {#passport}

---

**[!TAB Консоль управления]**

Чтобы авторизоваться, войдите в аккаунт Яндекса или Яндекс.Коннекта.

**[!TAB CLI]**

Чтобы авторизоваться, укажите ваш [OAuth-токен](oauth-token.md) при [инициализации профиля](../../../cli/quickstart.md#initialize) в CLI. Токен будет сохранен в конфигурации профиля и авторизация будет происходить автоматически.

## Авторизация сервисного аккаунта {#sa}

Cервисные аккаунты авторизуются через API Яндекс.Облака при помощи [IAM-токена](iam-token.md) и [авторизованных ключей](key.md), а в сервисах AWS-совместимым API при помощи [статических ключей доступа](access-key.md), например в [!KEYREF objstorage-name].
