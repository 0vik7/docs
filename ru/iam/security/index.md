# Управление доступом

Пользователь Яндекс.Облака может выполнять только те операции над ресурсами, которые разрешены назначенными ему ролями. Пока у пользователя нет никаких ролей, все операции ему запрещены.

Чтобы разрешить доступ к ресурсам сервиса Yandex Identity and Access Management (сервисным аккаунтам и ключам доступа к ним), назначьте пользователю нужные роли из приведенного ниже списка. Вы можете назначить пользователю роль на сам сервисный аккаунт, на каталог, в котором находится аккаунт, или на все облако — права доступа в Яндекс.Облаке наследуются.

Подробнее о наследовании ролей читайте в разделе [#T](../../resource-manager/concepts/resources-hierarchy.md#access-rights-inheritance) документации сервиса Yandex Resource Manager.

{% endnote %}

### Примитивные роли

Примитивные роли можно назначать на любой ресурс в любом сервисе.

#### viewer

Пользователь с ролью `viewer` может просматривать информацию о ресурсах, например получить список ключей доступа для сервисного аккаунта.

#### editor

Пользователь с ролью `editor` может управлять любыми ресурсами, например создать сервисный аккаунт или ключи доступа для него.

Помимо этого роль `editor` включает в себя все разрешения роли `viewer`.

#### admin

Пользователь с ролью `admin` может управлять правами доступа к ресурсам, например разрешить другим пользователям использовать сервисные аккаунты или просматривать информацию о них.

Помимо этого роль `admin` включает в себя все разрешения роли `editor`.
