# Безопасность и управление доступом

## Назначение ролей

Вы можете управлять правами доступа к сервисным аккаунтам и ключам доступа для них.

Вы можете назначить пользователю роль на сам сервисный аккаунт. Помимо прав доступа на сам сервисный аккаунт, наследуются права доступа на каталог и облако, в которых находится сервисный аккаунт.

#### См. также

- [[!TITLE]](../../resource-manager/operations/folder/set-access-bindings.md).
- [[!TITLE]](../../resource-manager/operations/cloud/set-access-bindings.md).


## Роли

В этом разделе описано, какие права доступа дает каждая роль в сервисе [!KEYREF iam-full-name].

[!INCLUDE [cloud-roles](../../_includes/cloud-roles.md)]

#### [!KEYREF roles-viewer]

Пользователь с ролью `[!KEYREF roles-viewer]` может просматривать информацию о ресурсах, например получить список ключей доступа для сервисного аккаунта.

#### [!KEYREF roles-editor]

Пользователь с ролью `[!KEYREF roles-editor]` может управлять ресурсами, например создать сервисный аккаунт или ключи доступа для него.

Помимо этого роль `[!KEYREF roles-editor]` включает в себя все разрешения ролей `[!KEYREF roles-viewer]`.

#### [!KEYREF roles-admin]

Пользователь с ролью `[!KEYREF roles-admin]` может управлять правами доступа к ресурсам, например разрешить другим пользователям использовать сервисные аккаунты или просматривать информацию о них.

Помимо этого роль `[!KEYREF roles-editor]` включает в себя все разрешения роли `[!KEYREF roles-editor]`.
