# Удалить сервисный аккаунт

> [!IMPORTANT]
> Удаление пользователя из облака — неотменяемая операция. Восстановить удаленный сервисный аккаунт невозможно, только [создать его заново](create.md).

---

**[!TAB Консоль управления]**

1. Выберите каталог.
2. Выберите вкладку **Сервисные аккаунты**.
3. Нажмите на ![](../../../_assets/dots.png) напротив сервисного аккаунта и выберите **Удалить сервисный аккаунт**.
4. Подтвердите удаление.

**[!TAB CLI]**

Удалите сервисный аккаунт по его имени или идентификатору. Поиск сервисного аккаунта осуществляется в каталоге по умолчанию.

```
yc iam service-account delete <SERVICE-ACCOUNT-NAME>|<SERVICE-ACCOUNT-ID>
```

Если сервисный аккаунт лежит в другом каталоге, вы можете указать каталог с помощью флагов `--folder-id` или `folder-name`.

```
yc iam service-account delete sa-name-to-delete \
    --folder-name yet-another-folder
```

**[!TAB API]**

Чтобы создать сервисный аккаунт, воспользуйтесь методом `Delete` для ресурса `ServiceAccount`.
