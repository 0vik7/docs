# Методы в APIДля управления ресурсами в API определен набор методов. Каждому gRPC-методу ставится в соответствие HTTP-глагол. Например, методу `List` соответствует глагол `GET`,  а методу `Create` — глагол `POST`. Соответствия HTTP-запросам задаются в описании методов, в   аннотации [google.api.http](https://github.com/googleapis/googleapis/blob/master/google/api/http.proto).В API есть два набора методов: _стандартные_ и _дополнительные_. [Стандартные методы](standard-methods.md) позволяют выполнять над ресурсами CRUD-операции: создавать, получать, изменять и удалять. Стандартные методы работают одинаково для всех API.  [Дополнительные методы](custom-methods.md) расширяют функциональность стандартных методов.  С их помощью можно выполнить такие операции, как подключить диск, запустить виртуальную   машину и др. Сигнатура дополнительных методов отличается от сигнатуры стандартных методов.> [!NOTE]>> Справочники gRPC API находятся в разработке и будут опубликованы в ближайшее время.
