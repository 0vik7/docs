# Взаимосвязь ресурсов сервиса [!KEYREF dataproc-name]

Сервис [!KEYREF dataproc-name] позволяет реализовать распределенное хранение и
обработку данных с помощью сервисов из экосистемы [Apache Hadoop](http://hadoop.apache.org).

Основная сущность сервиса — _кластер_, который объединяет все ресурсы, доступные Hadoop, — вычислительные мощности и хранилище.

Каждый кластер состоит из _подкластеров_. Подкластеры объединяют _хосты_, выполняющие одинаковые функции:

* Подкластеры для хранения или обработки данных (например, DataNode для HDFS или NodeManager для YARN).
* Главный подкластер с управляющим хостом (например, NameNode для HDFS или ResourceManager для YARN).

    > [!NOTE]
    >
    > На стадии Preview главный подкластер состоит не более чем из одного хоста.

Подкластеры каждого кластера должны находиться в одной облачной сети, в одной зоне
доступности Облака. [Подробнее о географии](../../overview/concepts/geo-scope.md).

Хосты в каждом подкластере создаются с вычислительной мощностью, соответствующей указанному _классу хостов_. Список доступных классов хостов и их характеристики см. в разделе [[!TITLE]](instance-types.md).

О сетевой конфигурации и сетевом доступе к кластеру см. раздел [[!TITLE]](network.md).
